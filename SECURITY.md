# Security Policy

We take the security of datapitfalls seriously. Thank you for helping keep the project and its users safe.

## Supported Versions

datapitfalls is in early development. Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

As the project matures and reaches a stable 1.0 release, this table will be updated to reflect the supported release lines.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them privately by email to:

📧 **security@dataliteracy.com**

When reporting, please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof-of-concept
- The affected version(s)
- Any suggested remediation, if you have one

## What to Expect

- **Acknowledgment:** We will acknowledge receipt of your report within **48 hours**.
- **Assessment:** We will provide an initial assessment of the report within **7 days**.
- **Resolution:** We will keep you informed of our progress as we work toward a fix, and we'll coordinate with you on disclosure timing.

We're grateful to security researchers and community members who report vulnerabilities responsibly. With your permission, we're happy to credit you once an issue is resolved.
